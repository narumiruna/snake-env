Level 1 — 基礎

貪食蛇遊戲與 Gymnasium 環境

在開始了解 AI 演算法之前，我們先搞清楚「環境」是什麼。 這個專案使用 Gymnasium（前身為 OpenAI Gym）——一個讓 AI 代理與遊戲互動的標準介面。

🎮 遊戲規則

  • 棋盤大小預設為 20 × 20（400 格）
  • 蛇從 (0, 0) 出發，每步可選擇 上 / 右 / 下 / 左 四個方向
  • 吃到蘋果 → 蛇身增長 1 格，得 1 分；蘋果隨機重生
  • 撞牆或咬到自己 → 遊戲結束（失敗）
  • 蛇填滿全部 400 格 → 遊戲結束（完美勝利）

（示範：AI 控制的 4×4 棋盤）

📐 觀察空間（Observation Space）

每一步，環境提供給 AI 兩個資訊：

board
一個 N × N 的矩陣，每格的值代表：
0 = 空、1 = 頭、2 = 身體、3 = 蘋果
direction
目前蛇頭的行進方向
0=上、1=右、2=下、3=左

🎯 動作空間（Action Space）

AI 每步必須回傳一個整數 0–3（上 / 右 / 下 / 左）。 注意：反方向會被忽略（蛇不能直接掉頭），保持原方向移動。

💰 獎勵設計（Reward）

事件獎勵
吃到蘋果+1.0
完美勝利（填滿棋盤）+10.0
撞牆 / 咬自己−1.0
超過步數上限（沒吃到蘋果）−0.5
普通移動−0.01（生存懲罰，避免繞圈）
💡 為什麼要有生存懲罰？
若每步獎勵為 0，AI 可能學到「無所事事的繞圈策略」。微小的 −0.01 懲罰迫使它盡快吃蘋果。

📦 對應程式碼

# src/snake_env/env.py
class SnakeEnv(gym.Env):
    def step(self, action: int):
        # 移動蛇頭
        new_head = self.head.move(self.direction)

        if self._out_of_bounds(new_head):
            return ..., -1.0, True, ...   # 撞牆

        if new_head in occupied:
            return ..., -1.0, True, ...   # 咬自己

        if will_grow and self.won:
            return ..., 10.0, True, ...   # 完美勝利

        reward = 1.0 if will_grow else -0.01
        return ..., reward, False, ...
Level 2 — 進階

漢米頓環（Hamiltonian Cycle）

純靠 RL 訓練要讓蛇填滿 400 格幾乎不可能——搜索空間太大。 這個專案用了一個「數學保證絕不輸」的基礎策略：漢米頓環。

什麼是漢米頓環？

在圖論中，漢米頓環（Hamiltonian Cycle）是一條路徑， 恰好經過圖中每個頂點一次，最後回到起點。

把棋盤想像成一張網格圖，每個格子是一個頂點，相鄰格子之間有邊。 如果我們找到一條漢米頓環，蛇只要沿著這條路走，就能：

  • 永遠不會撞牆（環在棋盤內）
  • 永遠不會咬到自己（每格只走一次）
  • 最終一定會遇到所有格子（包含蘋果）
💡 重要前提
棋盤大小必須是 偶數 × 偶數（如 20×20），這樣才保證漢米頓環存在。 這也是為什麼 SnakeEnv 強制 size % 2 == 0。

這個專案如何建構漢米頓環？

程式碼使用一個固定的幾何規律，對任意偶數 N 都有效：

6×6 棋盤的漢米頓環（箭頭表示行進順序）

建構邏輯（程式碼簡述）：

  1. 第一列（row 0）從左到右：(0,0) → (1,0) → ... → (N-1, 0)
  2. 對右側各列（row 1 到 N-1），奇偶列方向相反（蛇形走法）
  3. 最後沿最左行（x=0）往上回到起點
# src/snake_env/models/hamiltonian.py
def hamiltonian_cycle(size: int) -> list[Pos]:
    cells = [Pos(x, 0) for x in range(size)]  # 第一列
    for x in range(size - 1, 0, -1):
        offset = size - 1 - x
        if offset % 2 == 0:
            cells.extend(Pos(x, y) for y in range(1, size))
        else:
            cells.extend(Pos(x, y) for y in range(size - 1, 0, -1))
    cells.extend(Pos(0, y) for y in range(size - 1, 0, -1))  # 左側回頭
    return cells

HamiltonianPolicy 如何使用？

建構環之後，策略非常簡單：

  1. 預先計算每個位置的「下一格」（next_pos 字典）
  2. 每步只需查表：我在哪裡？→ 下一格是哪裡？→ 要往哪個方向走？
class HamiltonianPolicy(SnakePolicy):
    def act(self, env: SnakeEnv) → int:
        return self._action_toward(env.head, self.next_pos[env.head])

環的距離（Cycle Distance）

每個格子在環中都有一個「序號」。環距離是從 A 出發，沿環方向到達 B 需要走幾步。 這個概念在後面的優化中非常重要：

cycle_distance(start, end) = (index[end] − index[start]) mod (N × N)

漢米頓策略的缺點

⚠️ 效率問題
漢米頓策略保證勝利，但非常浪費。 就算蘋果在蛇頭正前方，蛇也得繞完整個環才能走到那裡。 20×20 棋盤平均需要 ~39,629 步才能贏。

Level 3 — 進階

效率漢米頓捷徑策略

EfficientHamiltonianPolicy 繼承了漢米頓策略的「保證勝利」特性， 但加入了一個聰明的優化：在安全的時候走捷徑。

核心概念：安全捷徑

與其只能沿環走，現在蛇可以「跳過」環中的一段，直接往前移動—— 前提是這個捷徑不會讓蛇頭追上蛇尾（這樣才不會咬到自己）。

左：必須繞遠路的情況 ｜ 右：可以走捷徑的情況

安全判斷邏輯

對每個候選動作（移動到相鄰格 next_pos），演算法做以下安全檢查：

步驟 1
基本過濾
排除反方向、出界、自碰撞
步驟 2
計算前進距離
這一步在環上走了幾格？
步驟 3
安全上限
前進距離 ≤ 頭到尾的環距離
步驟 4
貪心選擇
選擇到蘋果環距離最短的動作

為什麼「前進距離 ≤ 頭到尾的環距離」是安全的？

把環想像成一條單向跑道。蛇頭在前，蛇尾在後。 蛇頭的位置永遠不能「追上」蛇尾——否則就咬到自己了。

用環上的序號：如果 index[head] < index[tail]， 蛇頭可以「跳躍」（走捷徑），只要跳完後蛇頭的序號仍 < 蛇尾序號，就是安全的。

蛇會長大（吃蘋果時）所以判斷稍有差異，will_grow 時安全上限更嚴格：

max_forward = tail_distance　（吃蘋果，蛇變長）
max_forward = tail_distance + 1　（普通移動，蛇尾跟著縮）

程式碼對照

# src/snake_env/models/efficient.py
def _safe_forward_actions(self, env) → list:
    tail_distance = self.cycle_distance(head, tail)

    for action in Action:
        # ... 基本過濾 ...

        forward_distance = self.cycle_distance(head, next_pos)
        max_forward = tail_distance if will_grow else tail_distance + 1

        if forward_distance >= max_forward:
            continue  # 不安全，跳過

        apple_distance = self.cycle_distance(next_pos, env.apple)
        candidates.append((apple_distance, -forward_distance, action))

    # 選 apple_distance 最小的（最快吃到蘋果）
    return min(candidates)

💡 設計哲學
捷徑策略不需要任何學習——它完全基於數學保證。 這讓它成為 Q-Learning 代理的完美「先驗知識（expert prior）」： 當 AI 還不確定時，就跟著捷徑策略走，不會犯致命錯誤。
Level 4 — 強化學習

表格式 Q-Learning 強化學習

現在進入真正的機器學習部分。雖然漢米頓捷徑策略很好， 但也許 AI 可以從自身的遊戲經驗學到更聰明的策略？ 這就是 Q-Learning 的用武之地。

強化學習基本概念

強化學習的核心問題是：代理如何在與環境互動中學習最佳行為策略？

觀察
代理觀察當前狀態 s
行動
依據策略選擇動作 a
反饋
環境給予獎勵 r
學習
更新策略，記住什麼好什麼壞

什麼是 Q 值（Q-Value）？

Q(s, a) 是「在狀態 s 下執行動作 a，預期可以獲得多少未來總獎勵」。 Q-Learning 的目標就是學出這張表。

Q(s, a) ← Q(s, a) + α × [ r + γ × maxa' Q(s', a') − Q(s, a) ]
符號意義本專案數值
α（alpha）學習率：每次更新的幅度0.35
γ（gamma）折現率：多重視未來的獎勵0.98
r即時獎勵−1 / −0.01 / +1 / +10
s, s'當前 / 下一個狀態蛇的完整狀態
a, a'當前 / 未來最佳動作上 / 右 / 下 / 左

「表格式」是什麼意思？

Q-Learning 有很多實作方式（如 DQN 用神經網路）。 「表格式（Tabular）」指的是直接用一個 Python 字典儲存所有狀態 → Q 值的對應， 而不用神經網路。 

# 每個狀態 → 一個長度為 4 的列表（對應4個動作）
q: defaultdict[tuple, list[float]] = defaultdict(
    lambda: [0.0, 0.0, 0.0, 0.0]
)
⚠️ 狀態空間爆炸
20×20 棋盤的狀態數量理論上天文數字般巨大（蛇可以有無數種形狀）。 純表格式 Q-Learning 很難覆蓋所有狀態，因此本專案的聰明之處在於 結合專家策略（expert prior），讓 AI 在未知狀態下有安全的預設行為。

專家策略先驗（Expert Prior）

這是本專案最有趣的設計之一。TabularQAgent 在選擇動作時的邏輯是：

def act(self, env, *, greedy=True, epsilon=0.0):
    state = self.state(env)
    expert_action = self.expert.act(env)  # 效率漢米頓策略

    values = self.q.get(state)
    if not values or max(values) <= 0.0:
        return expert_action  # 還沒學過這個狀態 → 信任專家

    # 學過了 → 選 Q 值最高的動作（同分時偏好專家動作）
    return max(range(4), key=lambda a: (values[a], a == expert_action))

這意味著：

  • 🆕 新狀態：直接跟著效率漢米頓策略走（不會犯致命錯誤）
  • 📚 已學狀態：選 Q 值最高的動作，若同分則偏好專家意見

訓練過程：模仿學習（Imitation Learning）

本專案的訓練有一個特別之處：訓練時讓專家操作，AI 觀察並學習 （而不是 AI 隨機探索）。這叫做「行為複製（Behavior Cloning）」或「模仿學習」。 

for episode in range(config.episodes):
    while not done:
        state = agent.state(env)
        action = agent.expert.act(env)  # 由「專家」（效率漢米頓）選動作
        _, reward, done, _, info = env.step(action)

        shaped_reward = reward + (0.25 if reward > 0 else 0.0) - 0.001
        agent.learn_step(state, action, shaped_reward, next_state, done)
        agent.reinforce_expert_action(state, action, reward=0.1)

reinforce_expert_action 確保專家選過的動作 Q 值不會變成負數， 使 AI 永遠不會「主動避開」好的動作。

訓練參數總覽

訓練超參數
  • 學習率 α = 0.35
  • 折現率 γ = 0.98
  • 預設訓練 3 個 episode
  • 隨機種子保證可重現
獎勵塑形（Reward Shaping）
  • 吃到蘋果額外 +0.25
  • 每步扣 −0.001（加速學習）
  • 保留原始 ±1 / ±10 獎勵
Level 5 — 評估指標

效率評分指標

我們如何判斷哪個 AI 模型更好？ 這個專案定義了一個「效率優先」的評分公式，讓模型比較有明確的依據。

設計哲學

直覺上，一個好的貪食蛇 AI 要：

  1. 首先：一定要贏（勝率 = 1.0）
  2. 其次：用更少的步數贏（越快越好）
  3. 參考：分數高低（通常和勝率一致）

效率分數公式

efficiency_score = win_rate × 1,000,000 + avg_score − avg_steps_to_win

分解來看：

項目說明示例值
win_rate × 1,000,000勝率佔絕對主導（權重是百萬）1.0 × 10⁶ = 1,000,000
+ avg_score加上平均得分（輔助）+399
− avg_steps_to_win減去贏局的平均步數（越少越好）−37,569
合計效率漢米頓策略的分數962,830
💡 為什麼用 ×1,000,000？
確保「勝率 0.99」的模型在任何情況下都比「勝率 1.0 但步數多」的模型評分低。 100 萬這個數字比任何合理的步數差值都大，所以勝率永遠是第一優先。

比較兩個策略

模型勝率平均贏局步數效率分數
HamiltonianPolicy 1.0 39,629 1,000,000 + 399 − 39,629 = 960,770
EfficientHamiltonianPolicy 1.0 37,569 1,000,000 + 399 − 37,569 = 962,830

效率漢米頓策略多 +2,060 分， 完全因為步數更少（勝率相同，分數相同）。

對應程式碼

# src/snake_env/metrics.py
efficiency_score = (win_rate * 1_000_000.0) + avg_score - avg_steps_to_win

🎓 學習路徑回顧

你現在已經了解了這個專案的完整技術棧：

  1. 環境設計（SnakeEnv） — 遊戲規則、觀察空間、動作空間、獎勵函數是 RL 的基礎。
  2. 漢米頓環（HamiltonianPolicy） — 用圖論保證「一定能贏」的確定性策略。
  3. 效率捷徑（EfficientHamiltonianPolicy） — 在安全前提下走捷徑，大幅減少步數。
  4. Q-Learning（TabularQAgent） — 結合模仿學習和 Q 值更新，以確定性先驗為後盾學習更優策略。
  5. 效率指標 — 用 win_rate × 10⁶ + avg_score − avg_steps_to_win 量化模型品質。
